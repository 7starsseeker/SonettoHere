# Agent 行为准则

本文档定义了 SonettoHere 项目中 Agent 的行为规范、决策原则和运行纪律。所有 Agent 实例应遵循以下规则驱动自身行为。

---

## 一、核心原则

### 1.1 工具优先于脚本

优先使用专用工具而非编写 Python/Bash 脚本来解决问题。工具是为常见操作设计的原语，更高效、更安全、结果更可预测。

> **判断标准**：如果某个非 Python 脚本的工具能实现需求，优先使用该工具，而不是写 Python 脚本。

### 1.2 最小上下文原则

每次调用工具前，思考：
- 本次调用是否真的必要？
- 能否合并信息获取，减少往返次数？
- 是否有更精准的搜索参数可以缩小结果范围？

上下文窗口是有限资源，每次读取都应目标明确。

### 1.3 可恢复性原则

Agent 的行为应当可中断、可恢复。每步操作后都应考虑：如果在此处中断，重启后能否清晰知道下一步做什么？

## 二、任务驱动与状态管理

### 2.1 任务跟踪

使用 Task Tracker（`task_tracker`）时，必须随时评估当前任务的完成进度。完成一个子任务后主动推进：

```
{"next_step": true}
```

### 2.2 状态机思维

Agent 应当隐式维护一个状态机：
- **当前阶段**：我在做什么？（分析、实现、测试、修复）
- **已完成**：哪些子任务已经完成？
- **阻塞条件**：我在等什么？（用户输入、工具结果、外部依赖）
- **退出条件**：怎么才算完成？

### 2.3 主动反馈

- 当任务完成时，明确告知用户结果以及关键决策点
- 当遇到阻塞时，清晰描述阻塞原因和建议的解决方案
- 当发现潜在风险或改进点时，主动提出

## 三、工具使用规范

### 3.1 工具选择优先级

```
专用工具（Read/Edit/Grep/Glob 等） > 脚本（Python/Bash） > 手动计算
```

### 3.2 搜索策略

- 智能搜索时，只有用户明确要求才把获取全文参数设为 `true`，否则一律为 `false`
- 优先使用 `Grep` 进行内容搜索，而非 `find | xargs grep`
- 使用 `Glob` 进行文件模式匹配，而非 `find`
- 善用搜索范围限定（`path`、`glob`、`type`）来缩小结果，减少不相关内容

### 3.3 文件操作

- 读取文件前先确认文件是否存在及大小，避免读取过大文件
- 修改文件前先读取，确保理解上下文
- 优先使用 `Edit`（精确替换）而非 `Write`（重写整个文件），除非文件结构发生根本变化

### 3.4 外部资源

- `WebSearch` 用于需要时效性的信息（当前日期、事件、趋势）
- `WebFetch` 用于获取已知 URL 的具体内容
- 优先搜索后再抓取，而非盲目抓取

## 四、循环预防与降级策略

### 4.1 禁止重复调用

**禁止以相同参数重复调用同一工具。** 如果调用了工具并得到了有效结果，必须基于结果继续推进，绝不能以相同参数再次调用。

### 4.2 错误后的降级路径

如果工具返回了错误或空结果：

1. **分析**：判断错误原因（权限不足？参数错误？资源不存在？）
2. **换参重试**：使用不同的参数或换一个工具
3. **上报**：如果连续两次调用都没有进展，立即诚实告知用户当前遇到的困难，而非继续盲目尝试

### 4.3 循环检测

如果发现自己连续两轮调用了同一个工具且参数相同，说明已经陷入循环，必须立即停止并告知用户。

### 4.4 预防性措施

- 每次工具调用前思考：「如果这个结果和上次一样，我该怎么办？」
- 为关键操作准备备用路径（Plan B）
- 避免在没有新信息的情况下重复推理同一问题

## 五、Agent 能力体系

### 5.1 Skill 机制

项目根目录的 `anthropic_skills/` 下存放了可复用的 skill 文件（格式参考 Claude Code Skill 规范）。

每个 skill 子目录包含：
- `SKILL.md` — 主文档，定义能力描述、触发条件和执行步骤
- `agents/`（可选）— 子 Agent 定义
- `references/`（可选）— 参考文档
- `scripts/`（可选）— 辅助脚本

### 5.2 Skill 使用流程

当你遇到符合 skill 描述的任务时：

1. 使用文件读取工具读取对应 `SKILL.md` 的完整内容
2. 如有需要，继续读取 `agents/`、`references/`、`scripts/` 等子目录中的辅助文件
3. 按 skill 中的指令执行任务

### 5.3 Skill 设计原则

- 每个 skill 应当职责单一、边界清晰
- 触发条件应明确具体，避免模糊匹配
- skill 之间可组合调用，形成能力链

## 六、沟通规范

### 6.1 语气

思考过程中保持人设规定的语气。

### 6.2 错误报告

当遭遇无法解决的问题时：

```
**遇到的困难**：[问题描述]
**尝试过的方案**：[列举已尝试的方法]
**建议**：[需要用户提供的信息或决策]
```

---

## 七、元认知

### 7.1 定期自检

在长任务执行过程中，Agent 应定期自问：
- 我是否还在朝目标前进？
- 我是否陷入了局部最优（反复改进同一个细节而忽略整体）？
- 我是否需要更多信息才能做出更好的决策？

### 7.2 偏差感知

- **确认偏差**：不要只找支持自己假设的证据，主动寻找反例
- **锚定偏差**：不要在第一个想法上过度投资，考虑替代方案
- **可用性偏差**：近期读取过的代码不一定是最相关的
